# Paperclip Setup

Paperclip is an agent orchestration framework that runs multiple agents with heartbeat loops, task routing, and inter-agent communication. Each agent gets a config directory with a `config.yaml` and supporting markdown files.

## Config mapping

| AgentCraft file | Paperclip equivalent | Where it goes |
|-----------------|---------------------|---------------|
| soul.md | soul.md | Agent config directory |
| agents.md | agents.md | Agent config directory |
| heartbeat.md | heartbeat.md | Agent config directory |
| identity.md | Mapped into config.yaml | Agent config directory |
| tools.md | tools.md | Agent config directory |
| config.yaml | Generated by wizard | Agent config directory root |

## Quick start

1. [Generate a config](https://agntcraft.co/create) — select "Paperclip" as framework
2. Download the ZIP
3. Extract into your Paperclip agent's config directory
4. Register the agent in Paperclip — it picks up the config on next heartbeat

## config.yaml

The wizard generates a `config.yaml` that Paperclip uses to manage the agent:

```yaml
name: my-agent
model: claude-opus
adapter: claude_local
heartbeat: 3600        # seconds between cycles
system_prompt: soul.md # path to persona file
project: my-project
```

Key fields:
- **heartbeat** — how often the agent's heartbeat loop fires (in seconds). Set to `0` for on-demand only.
- **adapter** — which model adapter to use (`claude_local`, `openai`, `gemini`, etc.)
- **system_prompt** — points to the soul.md file for persona loading

## Agent directory structure

```
agents/
  my-agent/
    config.yaml
    soul.md
    agents.md
    heartbeat.md
    tools.md
```

## Multi-agent teams

Paperclip excels at running multiple agents. Generate a config for each agent role, then set up routing rules in Paperclip's project config. AgentCraft's "Teams" category has pre-built multi-agent templates.
