# Paperclip Setup

Paperclip is an agent orchestration framework that runs multiple agents with heartbeat loops, task routing, and inter-agent communication. Each agent gets a config directory with a `config.yaml` and supporting markdown files.

## How wizard files integrate

The AgentCraft wizard generates 7 universal UPPERCASE files that complement Paperclip's own `config.yaml`. The wizard does not generate a config.yaml — you write that yourself for Paperclip-specific settings (model, adapter, heartbeat interval). The wizard files provide the agent's identity, persona, and operating manual alongside it.

| Wizard file | Where it goes |
|-------------|---------------|
| README.md | Agent config directory |
| SOUL.md | Agent config directory |
| AGENTS.md | Agent config directory |
| USER.md | Agent config directory |
| HEARTBEAT.md | Agent config directory |
| IDENTITY.md | Agent config directory |
| TOOLS.md | Agent config directory |

## Quick start

1. [Generate a config](https://agntcraft.co/create) — select "Paperclip" as framework
2. Download the ZIP
3. Extract into your Paperclip agent's config directory alongside your existing `config.yaml`
4. Register the agent in Paperclip — it picks up the config on next heartbeat

## Agent directory structure

```
agents/
  my-agent/
    config.yaml     ← Paperclip's native config (you write this)
    README.md       ← Wizard output — agent reads this first
    SOUL.md
    AGENTS.md
    USER.md
    HEARTBEAT.md
    IDENTITY.md
    TOOLS.md
```

The wizard files sit alongside Paperclip's own `config.yaml` — they complement it, not replace it.

## Multi-agent teams

Paperclip excels at running multiple agents. Generate a config for each agent role, then set up routing rules in Paperclip's project config. AgentCraft's "Teams" category has pre-built multi-agent templates.
