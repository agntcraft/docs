# Codex Setup

OpenAI Codex reads `AGENTS.md` files for project-level instructions. It layers global, project, and directory-level files to build a complete instruction set.

## How wizard files integrate

The AgentCraft wizard generates 7 universal UPPERCASE files that complement Codex's own `AGENTS.md`. The wizard does not generate a Codex-specific AGENTS.md — you write that yourself for project-specific instructions. The wizard files provide the agent's identity, persona, and operating manual alongside it.

| Wizard file | Where it goes |
|-------------|---------------|
| README.md | Project root |
| SOUL.md | Project root |
| AGENTS.md | Project root |
| USER.md | Project root |
| HEARTBEAT.md | Project root |
| IDENTITY.md | Project root |
| TOOLS.md | Project root |

## Quick start

1. [Generate a config](https://agntcraft.co/create) — select "Codex" as framework
2. Download the ZIP
3. Extract the wizard files into your project root alongside your existing Codex config
4. Run Codex — it reads the files automatically

## File precedence

Codex reads instruction files in this order, layering them:

1. **`~/.codex/AGENTS.md`** — global instructions for all projects
2. **Project root `AGENTS.md`** — project-specific instructions
3. **Current directory `AGENTS.md`** — directory-scoped overrides

Later files override earlier ones. Use global for your personal coding style, project-level for tech stack and architecture, and directory-level for module-specific rules.

## Project structure

```
my-project/
  AGENTS.md        ← Codex reads this
  src/
    api/
      AGENTS.md    ← Optional: directory-scoped overrides
    frontend/
      AGENTS.md    ← Optional: directory-scoped overrides
  package.json
```
